# 🔥 ROUND 1 – Core Backend

### 1️⃣ What is middleware?

Middleware is a function that executes between request and response in backend.  
Used for authentication, logging, validation, error handling.

---

### 2️⃣ What is JWT?

JSON Web Token is a compact token used for authentication.  
It contains Header, Payload, Signature.  
Server verifies signature instead of storing session.

---

### 3️⃣ What is hashing?

Hashing converts data into fixed-length string using algorithm (SHA-256, bcrypt).  
Used for password storage.

---

### 4️⃣ Why not store plain passwords?

If DB leaks, attackers get real passwords.  
Instead use hashing + salt (bcrypt/argon2).

---

### 5️⃣ What is CORS?

Browser security mechanism blocking cross-origin requests unless allowed via headers.

---

# 🔥 ROUND 2 – SQL

### 6️⃣ Difference between WHERE and HAVING?

WHERE filters rows before GROUP BY.  
HAVING filters aggregated results after GROUP BY.

---

### 7️⃣ What is foreign key?

Column that references primary key of another table.  
Maintains referential integrity.

---

### 8️⃣ What is indexing?

Database optimization technique to speed up searches using data structure (B-tree).

---

### 9️⃣ What is normalization?

Organizing tables to reduce redundancy (1NF, 2NF, 3NF).

---

### 🔟 What is ACID?

Atomicity, Consistency, Isolation, Durability – guarantees reliable transactions.

---

# 🔥 ROUND 3 – System & OS

### 1️⃣1️⃣ What is deadlock?

Situation where two processes wait for each other indefinitely for resources.

---

### 1️⃣2️⃣ What is race condition?

When multiple threads access shared resource simultaneously causing unpredictable result.

---

### 1️⃣3️⃣ Difference between process and thread?

Process → separate memory  
Thread → shared memory inside process

---

### 1️⃣4️⃣ What is load balancer?

Distributes incoming traffic across multiple servers to improve scalability and availability.

---

### 1️⃣5️⃣ What is caching?

Storing frequently accessed data in memory (Redis) to reduce DB load.

---

# 🔥 ROUND 4 – REST & APIs

### 1️⃣6️⃣ What is idempotent API?

API that produces same result when called multiple times (GET, PUT, DELETE).

---

### 1️⃣7️⃣ Difference between PUT and PATCH?

PUT replaces entire resource.  
PATCH updates specific fields.

---

### 1️⃣8️⃣ What is stateless API?

Server does not store session. Each request carries complete info.

---

### 1️⃣9️⃣ What is 404 vs 401?

404 → Not found  
401 → Unauthorized (authentication required)

---

### 2️⃣0️⃣ What is 500 error?

Internal server error – server crashed or unhandled exception.

---
